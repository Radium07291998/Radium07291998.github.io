2022/10/23

1. Background

User interface (UI) is the medium between the game and the player. It realizes the conversion between the game meta information and user readable information. The user interface is an essential component in games. It is generally called UI in Unity3D and UMG in UE4.

So, how is the game UI designed and finally presented to the players?
Here is the working-flow for most mobile game project:
Game product manager. Responsible for planning and defining the information the game needs to display.
UX. Responsible for the interaction design.
Game art 2D design. Responsible for UI design.
UI refactor. Responsible for restoring UI (UMG) in game engine using UI design.
Game Development Engineer. Responsible for building the connection between game logic code of and UI (UMG).
The above roles accept the output from the previous person and then produce the results to send to next one.

By focusing on Stage 3 through Stage 4, we find that this process is redundant.

  1. We need to have an enormous workforce in the UI refractor process.
  2. We need to keep an up-to-date list of existing artistic resources, so that our project does not become cumbersome.
  3. The refactored user interface may sometimes be inconsistent with the original design of the user interface.
